Semantic HTML5 Structure

Semantic HTML tags clearly describe their meaning to both the browser and developer. They make web pages more informative and adaptable, allowing browsers and search engines to better interpret content.

Key Semantic Elements

  • <header> - Introductory content or navigational aids
  • <nav> - Navigation links
  • <main> - Dominant content of the document
  • <section> - Thematic grouping of content
  • <article> - Self-contained composition
  • <aside> - Content indirectly related to the main content
  • <footer> - Footer for its nearest sectioning content

Using semantic elements improves document structure, enhances accessibility, and helps search engines understand your content.

Web Accessibility Fundamentals

Web accessibility ensures that people with disabilities can perceive, understand, navigate, and interact with websites and tools.

Essential Accessibility Practices

  • Provide text alternatives for non-text content
  • Create content that can be presented in different ways
  • Make all functionality available from a keyboard
  • Help users navigate and find content
  • Make text readable and understandable

Properly using HTML5 semantic elements is one of the easiest ways to improve accessibility, as screen readers can better interpret page structure and content.

SEO Basics

Search Engine Optimization (SEO) is the practice of increasing the quantity and quality of traffic to your website through organic search engine results.

Key On-Page SEO Factors

  • Quality content that answers searchers' questions
  • Proper use of HTML headings (H1, H2, H3, etc.)
  • Descriptive title tags and meta descriptions
  • Semantic HTML structure
  • Image optimization with alt text
  • Mobile-friendly design
  • Fast loading times

By combining semantic HTML with accessibility best practices, you create a strong foundation for SEO success.

HTML5 Code Examples

Semantic Structure

<header>
  <h1>Page Title</h1>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>
</main>

<footer>
  <p>Copyright Info</p>
</footer>

Accessible Form

<form>
  <div>
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" 
           aria-describedby="username-help" required>
    <div id="username-help">Enter your username</div>
  </div>
  
  <button type="submit">Submit</button>
</form>
